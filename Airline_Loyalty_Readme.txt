## Project Summary

This project builds an end-to-end early-warning system for airline loyalty churn, using real data on ~16,700 Canadian loyalty members (2017–2018). It defines a behavioral "Silent Churn" signal, validates it against the dataset's own formal cancellation records, segments members on both behavioral and demographic dimensions, and trains a calibrated XGBoost model to score churn risk — feeding a Streamlit dashboard with segment-specific retention playbooks.

## Key Findings

1. **A validated churn definition.** "Silent Churn" (flew in 2017, zero flights in 2018) is cross-checked against formal cancellation records: **92.4% of members flagged as Silent Churn go on to formally cancel** — confirming it as a genuine leading indicator, not an arbitrary proxy.

2. **A clear, high-risk segment.** 5 segments were built using both behavioral *and* demographic signals (salary, education, marital status, card tier), with cluster count selected via silhouette score and stability-tested across 5 random seeds — not assumed. **At-Risk Infrequent Flyers (3,285 members) show a 10.6% churn rate, roughly 5× every other segment**, driven by inconsistent monthly activity, not income or marital status.

3. **A real data quality finding.** 20 records had negative salary values (a data entry error) — explicitly identified and corrected rather than silently averaged into the model.

4. **The model is calibrated, not just accurate.** 0.77 out-of-sample ROC-AUC, with probability calibration explicitly checked — a real 0.234 calibration gap was found and corrected to 0.0085 via isotonic regression before any score was used for targeting.

5. **A corrected column reference.** The original data dictionary specifies `Total Flights`, not `Flights Booked` — this was verified against the real files before building any pipeline logic on top of it.

## How to Run

### 1. Set up the folder
Place all files in a single folder (no subfolders — relative paths are used throughout):
```
Airline-Loyalty-Retention/
├── Airline_Loyalty_Intelligence.ipynb
├── loyalty_dashboard.py
├── Customer_Loyalty_History.csv
├── Customer_Flight_Activity.csv
├── Calendar.csv
├── Airline_Loyalty_Data_Dictionary.csv
└── scored_members.csv   (generated by step 2 — not present until you run the notebook)
```

### 2. Run the ML pipeline
Open `Airline_Loyalty_Intelligence.ipynb` in Jupyter, VS Code, or Colab. Run all cells sequentially (Runtime → Run all). This will:
- Load and clean the real data
- Build and cross-validate the Silent Churn definition
- Segment members (behavioral + demographic)
- Train and calibrate the XGBoost model
- Export `scored_members.csv`

Takes roughly 1–2 minutes to run end-to-end.

### 3. Launch the dashboard
```bash
pip install streamlit pandas numpy scikit-learn xgboost
python -m streamlit run loyalty_dashboard.py
```
This opens the dashboard in your browser (usually `http://localhost:8501`). Use the sidebar to filter by minimum churn risk and by strategic segment — selecting a specific segment shows its tailored playbook; "All Segments" shows a summary of all 5.

**Note:** the dashboard requires `scored_members.csv` to exist — if you see a "file not found" error, run the notebook (step 2) first.

## Methodology Notes

This project deliberately documents what it tested and found null or modest results for (enrollment channel showed zero variation; geography showed a modest but secondary signal), alongside its strong findings (the behavioral segmentation, the calibration fix). Every quantitative claim in the technical report is traceable to a specific notebook cell and was verified by re-running the pipeline before being written down.
